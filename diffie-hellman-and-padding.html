Diffie Hellman Key Exchange

In this section we want to introduce you to a key exchange protocol called Diffie Hellman.

Diffie-Hellman Key Exchange is an algorithm that uses a public channel to send a shared secret value/key between two users.

Motivation: Suppose two users, Bob and Alice, are trying to send a private message to one another. They want to establish some shared secret value between each other. However, attackers may be listening in on the messages sent between Bob and Alice. So, the question is, how can Bob and Alice agree on a value, but an attacker listening on the exchanged messages not be able to figure out the shared value? Well, that is where the Diffie Hellman Key Exchange protocol comes in.

General Idea: Bob and Alice have to use a math function that is easy to solve in one direction and difficult to solve in the reverse direction to agree on a shared secret value (a key). Bob and Alice first agree on some initial values and a math function that are published for all to see. Then, both Alice and Bob choose a secret value and substitute it into the function and then send their total value to the other. Then, with the public total they receive, along with the private value, they are able to agree on a shared secret key. A more in depth discussion on how exactly this works will be broken down in the video below.

Man in the Middle Attack: As we said above, an attacker is able to listen to all the messages that are being sent between Alice and Bob. So, the question is, could the attacker pretend to be Bob and end up sharing a secret key with Alice? Likewise, could the attacker pretend to be Alice and end up sharing a secret key with Bob? The answer is yes, and this is the essence of a man in the middle attack for Diffie Hellman. Then, with this secret key, the attacker could talk to both Alice and Bob and neither would know for sure that this attack is happening. Padding

In this section we want to introduce you to padding.

Padding is the process of adding text to a message in order to encrypt/hide its contents.

    There are three methods of padding
  1. Salt
  2. Pepper
  3. Nonce

Salting works by adding a unique string of text to the end of a password and then hashing the password plus salt.

A Pepperis almost exactly the same as a salt, with the exception that it is not to be stored in the database.

The term Nonce is used for a number that is only used once. Nonces work similarly to that of salts and peppers and are commonly used in message authentication codes.
