Section 1 - Why Do You Need Anti-Virus Software?

A virus is a type of malware that spreads to and infects other systems. They can steal passwords and data, corrupt files, and cause damage to your system. To avoid this problem, anti-virus software is needed to detect viruses and prevent them from harming your system. Below are summaries and videos of the detection techniques that anti-virus software implement. Remember, anti-virus software does not perfectly protect your system from threats. The user needs to be smart when browsing the web.

Section 2 - Signature-Based Detection

Signature Based Detection is one of the main techniques in virus detection software. The idea behind it is simple: look for threats that look like previous threats. When a new type of threat appears, a profile, or a signature, is created for this type of threat. This way, a system can find threats that are similar to ones that you already know about. The downside to this method is that there needs to be an initial threat that enters your system before you can have a signature on it. In other words, the signatures that your system has can’t identify unknown threats. At the same time, attackers may be able to change what threats “look like” through polymorphism, as they spread from system to system. This means that a signature-based detection algorithm may not always be useful.

Section 3 - Behavioral-Based Detection

Behavioral-Based Detection is similar to signature-based detection. Instead of looking at signatures, this detection algorithm looks at how a program intends to act upon a system. A program entering a system will be analyzed for any suspicious activity. If the program acts in an abnormal way, it will be labeled malicious. A benefit of this detection algorithm is that it can help to uncover unknown threats; a downside of this algorithm is that some objects may be incorrectly labeled as malicious when they are not.

Video on Behavioral-Based Detection

Section 4 - Heuristic-Based Detection

Heuristic-Based Detection looks for threats by searching for specific aspects., such as potentially malicious commands or instructions. Each program is graded based off how dangerous it can be. After a certain threshold, a program is deemed as malicious. The more malicious attributes, such as malicious commands, a program has, the higher its danger. In this algorithm, file analysis, file emulation, and genetic signature detection are used. File analysis looks at the intent of a program; file emulation tests a program in a controlled environment to see what it will do; genetic signature detection compares programs against virus signatures.
